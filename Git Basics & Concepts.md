# Git Basics & Concepts

## 📌 What is Git?
Git is a distributed version control system used to track changes in source code during software development.

## 📌 Why Use Git?
- Track changes
- Collaborate with team
- Maintain version history
- Restore previous versions

## 📌 Git vs GitHub
- Git → Version control tool (local)
- GitHub → Cloud platform to host Git repositories

## 📌 Key Concepts

### 🔹 Repository (Repo)
A project folder tracked by Git.

### 🔹 Commit
A snapshot of your changes.

### 🔹 Branch
A separate line of development.

### 🔹 Merge
Combining changes from different branches.

### 🔹 Clone
Copying a remote repository to your local system.

### 🔹 Pull
Fetching and updating code from remote repo.

### 🔹 Push
Uploading local changes to GitHub.

## 📌 Git Workflow
1. Working Directory
2. Staging Area
3. Repository

## 📌 Basic Flow Diagram
Working Directory → Staging Area → Commit → Remote Repo

## 📌 Important Notes
- Always commit with meaningful messages
- Pull before push to avoid conflicts
- Use branches for new features
